Design to Web

11.03.2016

Why is responsiveness important?

More and more, people are viewing websites on their mobile devices. This means that websites need to be able to respond to the screen size of the device used and adjust their layout accordingly. If not, the website may not display correctly and the viewer may leave the site, never to return!

What is mobile first design, and why it's important?

Mobile first design, as opposed to desktop first, is the process by which a website is styled with CSS to fit on a mobile devices. From there, additional CSS can be applied for larger screen sizes. This means that smaller devices are not bogged down processing a whole bunch of extra code, that is only applicable to larger devices -as is the case with desktop first design.

What are frameworks, and their pros and cons?

A CSS framework is a bunch of pre-written CSS code that works to streamline the design process
A big pro is that they can save the developer time. A con is that a lot of the work is already done, so you/the developer may loose out on a valuable learning experience.

What is a wireframe and why do we use them?

Wireframes are the starting point in the website design process. This is the basic layout or hierarchy of information that the website will have. There is no style or content involved in the wireframe. This allows the developer to quickly plan out the different elements of a website

What aspects of your wireframes did you find difficult to implement, and why?

I found sticking to the wireframe the most difficult. I realised that what i had outlined in my wireframes did not workout in practice, so i made some changes. I also found it difficult make my site index element appear and disappear, it took a while to get that to work.
